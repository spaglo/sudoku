## Что нужно сделать

### 1. Проверить, что всё работает у тебя

```pwsh
cd <папка, куда распаковал>
dotnet build                     # 0 ошибок 0 варнингов
dotnet test                      # должно быть 24/24 зелёных
```

Нужен **.NET 8 SDK** (`dotnet --version` → 8.0.x или 10.0.x подойдёт).

### 2. Создать репозиторий на GitHub

1. Зайди на github.com → New repository.
2. Имя: `SudokuPair` (любое — главное согласуй со Студентом Б).
3. **Private**.
4. **Без** README и .gitignore — у нас свои.
5. Create repository.

### 3. Залить эту папку в репозиторий

В консоли в папке проекта:

```pwsh
git init
git add .
git config user.name "<Твоё имя>"
git config user.email "<твой email от GitHub>"
git commit -m "feat: initial scaffolding + Sudoku engine + tests"
git branch -M main
git remote add origin https://github.com/<твой логин>/SudokuPair.git
git push -u origin main
```

После этого репозиторий на github.com будет содержать всю твою часть.

### 4. Дать доступ Студенту Б

1. На GitHub: репозиторий → **Settings** → **Collaborators** → **Add people**.
2. Введи GitHub-логин Б, отправь invite.
3. Б примет invite и сможет коммитить.

### 5. Сообщить Б, что готово

Скажи: «движок готов в `<ссылка на репо>`, начинай делать `Sudoku.UI/`».

С этого момента ты ждёшь его пуш с UI. Когда придёт — `git pull`, и в Visual Studio появится третий проект.

## Несколько коммитов вместо одного (опционально, но красиво)

Если хочется, чтобы история выглядела как поэтапная разработка, разбей первый коммит на несколько перед пушем. Например:

```pwsh
git init
git config user.name "<Твоё имя>"
git config user.email "<твой email>"

git add .gitignore README.md СТАРТ.md Sudoku.sln
git commit -m "chore: init solution and scaffolding"

git add Sudoku.Engine/Sudoku.Engine.csproj \
        Sudoku.Engine/Difficulty.cs \
        Sudoku.Engine/MoveResult.cs \
        Sudoku.Engine/CellInfo.cs \
        Sudoku.Engine/MoveAppliedEventArgs.cs
git commit -m "feat(engine): public API contracts (Difficulty, MoveResult, CellInfo)"

git add Sudoku.Engine/Board.cs
git commit -m "feat(engine): Board with row/column/block validation"

git add Sudoku.Engine/BoardSolver.cs
git commit -m "feat(engine): backtracking solver with MRV heuristic"

git add Sudoku.Engine/BoardGenerator.cs
git commit -m "feat(engine): puzzle generator with uniqueness check"

git add Sudoku.Engine/SudokuGame.cs
git commit -m "feat(engine): SudokuGame facade — lives, hints, timer, events"

git add Sudoku.Engine.Tests/
git commit -m "test(engine): 24 NUnit tests covering Board / Solver / Generator / Game"

git branch -M main
git remote add origin https://github.com/<твой логин>/SudokuPair.git
git push -u origin main
```

## Дальше — что делать, когда Б закончит

1. `git pull` — приедут файлы `Sudoku.UI/` + обновлённый `Sudoku.sln`.
2. Открой `Sudoku.sln` в Visual Studio — теперь три проекта в Solution Explorer.
3. ПКМ по `Sudoku.UI` → **Set as Startup Project** → F5 → играй партию.

Если будут конфликты при `git pull` (скорее всего в `Sudoku.sln` и `README.md`) — пиши, помогу.

## Что НЕ нужно делать

- ❌ Трогать папку `Sudoku.UI/` — это часть Б.
- ❌ Менять публичные сигнатуры классов `SudokuGame`, `CellInfo`, `MoveResult`, `Difficulty` — Б на них опирается. Если очень нужно — сначала созвон.
- ❌ Коммитить `bin/`, `obj/`, `.vs/` — `.gitignore` уже их игнорирует.

Удачи! 🎮
